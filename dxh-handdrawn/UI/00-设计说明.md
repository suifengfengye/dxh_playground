# 小红书手动绘画视频生成系统 - UI 设计说明

## 1. 设计目标

- 面向内容创作者，快速把一句提示词转成“手动绘画过程视频”。
- 首页只保留一条最短操作路径：输入提示词 -> 点击开始生成 -> 在 3 个预览板块中查看结果。
- 视觉上强调“创作感、工具感、视频感”，避免传统后台过重的管理台气质。

## 2. 页面结构

- 顶部：品牌区、项目状态、导出入口占位。
- 左侧主操作区：提示词输入、风格标签、开始生成按钮、进度提示。
- 右侧预览区：3 个并列但层级不同的预览板块。
- 底部辅助区：生成参数摘要、作品说明、使用提示。

## 3. 三个预览板块定义

### 预览板块 A：脚本拆解预览

- 展示系统如何把提示词拆成镜头脚本。
- 内容包括：主题、分镜顺序、旁白草稿、绘画步骤摘要。
- 价值：让用户先确认“生成方向是否对”。

### 预览板块 B：画面分镜预览

- 展示每一段视频可能出现的关键画面卡片。
- 每张卡片包含：镜头编号、构图说明、绘制重点、镜头时长。
- 价值：让用户快速判断画面节奏和视觉风格。

### 预览板块 C：成片时间轴预览

- 展示最终视频的拼接效果和进度节点。
- 内容包括：封面、开场、绘制过程、收尾文案、BGM 节点。
- 价值：让用户判断“是否适合小红书发布”。

## 4. 核心交互

- 输入框支持长提示词，默认 3 行高度，可扩展。
- “开始生成”按钮是主 CTA，固定高识别度。
- 生成中状态使用顶部进度条 + 各预览区骨架屏联动。
- 三个预览区都保留“刷新此区”与“展开查看”入口占位。

## 5. 视觉方向

- 风格关键词：创作型工作台、深色视频编辑氛围、温暖强调色、柔和卡片、轻量专业感。
- 主背景：深蓝黑色，突出视频和画面内容。
- 强调色：玫红偏红，突出“开始生成”和关键状态。
- 辅助色：靛蓝、浅金橙，用于标签、步骤、进度信息。
- 质感：柔和阴影 + 轻玻璃感边框，不做厚重拟物。

## 7. 字体建议

- 标题：`Fredoka`
- 正文：`Nunito`
- 数字和辅助信息：沿用 `Nunito`

## 8. 布局建议

- Desktop：左 360 操作区 + 右侧自适应预览区。
- Tablet：上方操作区，下方 3 块预览 2 + 1 排列。
- Mobile：纵向单列，先操作后预览，预览区使用横向滑动卡片。
